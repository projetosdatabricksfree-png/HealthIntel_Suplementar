# HealthIntel Suplementar v3.0.0 — Fase 4 Concluída

**Versão:** `v3.0.0`  
**Data:** 2026-04-24  
**Escopo:** Fase 4 — Prata completa, ingestão real SIB/CADOP, Gold curado, `consumo_ans`, catálogo e ELT universal ANS.  
**Status:** Concluída  
**Release Candidate:** Encerrado  
**Tag final:** `v3.0.0`

## 1. Resumo Executivo

A versão `v3.0.0` fecha oficialmente a Fase 4 do HealthIntel Suplementar.
O produto permanece API-first e DaaS: entrega dados públicos ANS curados via `api_ans`
e `consumo_ans`, sem BI interno embutido.

Esta release entrega:

- camada Prata completa para os endpoints registrados em `PRATA_DATASETS`;
- ingestão real SIB/CADOP com landing zone, controle de lote e idempotência;
- marts Gold em `nucleo_ans`;
- camada `consumo_ans` para entrega PostgreSQL direta ao cliente;
- role `healthintel_cliente_reader` com isolamento das camadas internas;
- catálogo de dados para clientes;
- ELT universal ANS orientada por catálogo para discovery, download, manifesto e Bronze genérica.

## 4. Evidências Operacionais

- `plataforma.lote_ingestao` registrou cargas SIB/CADOP com `hash_arquivo`.
- A segunda execução SIB/CADOP registrou `ignorado_duplicata`.
- A validação SQL de duplicidade retornou 0 linhas para mais de um sucesso por `hash_arquivo`.
- A role `healthintel_cliente_reader` é `NOLOGIN`.
- A role cliente acessa `consumo_ans`.
- A role cliente recebe bloqueio nas camadas internas `bruto_ans`, `stg_ans`, `int_ans`, `nucleo_ans` e `plataforma`.
